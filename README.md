## рҹҺҜ The Problem India Can't Ignore

India conducts **over 2 crore competitive examinations annually** вҖ” determining futures, careers, and livelihoods for millions of students from every corner of the country. These examinations are meant to be the great equalizer. But that promise has been systematically broken.

| Year | Incident | Impact |
|------|----------|--------|
| 2024 | **NEET-UG** вҖ” paper leaked by Hazaribagh syndicate, circulated via WhatsApp | 24 lakh aspirants affected, 1,563 re-tested, Supreme Court intervention |
| 2024 | **UGC-NET** вҖ” cancelled the day after conduction due to confirmed malpractice | Entire examination nullified |
| 2024 | **BPSC** вҖ” question paper circulated on Telegram 1 hour before start | State-wide re-examination ordered |
| 2024 | **SSC CGL** вҖ” Tier-1 postponed amid leak allegations, CBI investigation ongoing | 70,000+ posts affected |
| 2017 | **SSC Scam** вҖ” systematic corruption exposed within India's largest recruitment body | Protests across 20+ states |
| 2013вҖ“2023 | **Vyapam** вҖ” exam manipulation operated for years inside the system | 55+ deaths, hundreds arrested |

**Over 70 confirmed paper leaks across 15 states since 2015 have disrupted the academic futures of approximately 1.7 crore students.**

### Root Cause Analysis

Every single one of these scandals shares one vulnerability:

> **One person with system access had enough power to act alone.**

A single corrupt official. A single compromised database. A single disloyal invigilator. Existing solutions вҖ” encrypted PDFs, sealed envelopes, biometric attendance, CCTV вҖ” all fail because they address symptoms, not architecture. In every existing system, **one party holds complete control**. Pariksha eliminates that single point of failure entirely.

## рҹӣЎпёҸ TriShield Vault вҖ” The Core Innovation

> *Concept and Innovation by Ketan Rawat, B.Tech Final Year, ECE Department, NIT Jamshedpur*
> *Not implemented by any examination authority in India or internationally.*

### What is TriShield?

TriShield Vault is the world's first **three-party cryptographic paper custody system** for examinations. It makes paper leaks **mathematically impossible**, not just policy-prohibited.

### The Six-Step Paper Lifecycle

```
STEP 1 в”Җв”Җв–ә Institute types questions
           в””в”Җв–ә Each question encrypts in-browser (Web Crypto API)
               as user types. Padlock animation. SHA-256 updates live.

STEP 2 в”Җв”Җв–ә Institute submits Edit Request (if needed)
           в””в”Җв–ә Request row written to paper_edit_requests table
               Admin notified via Supabase Realtime instantly.

STEP 3 в”Җв”Җв–ә Admin reviews and Approves Edit Window
           в””в”Җв–ә edit_window_open = true, 15-min timer starts
               BOTH institute AND admin screens show timer simultaneously.

STEP 4 в”Җв”Җв–ә Institute enters passkey вҶ’ questions decrypt
           в””в”Җв–ә PBKDF2 derives Fragment 1 client-side
               Server provides Fragment 2. Never stored. Never transmitted raw.

STEP 5 в”Җв”Җв–ә Edit made вҶ’ Re-encrypt вҶ’ New hash generated
           в””в”Җв–ә Old hash vs New hash displayed side-by-side
               Diff shows exactly what changed. Written to audit_log.

STEP 6 в”Җв”Җв–ә Vault resealed. Audit trail complete.
           в””в”Җв–ә SHA-256 monitoring resumes every 60 seconds
               Green PAPER INTEGRITY VERIFIED banner.
```

### Why No Existing System Does This

| Attack Vector | Existing Platforms | Pariksha TriShield |
|---------------|-------------------|-------------------|
| Corrupt institute employee modifies paper | вң… Possible вҖ” they have full access | вқҢ Impossible вҖ” needs admin approval + audit |
| Corrupt admin reads/modifies paper | вң… Possible вҖ” admin has DB access | вқҢ Impossible вҖ” no institute passkey |
| Database hacker steals questions | вң… Gets plaintext or weakly encrypted data | вқҢ Gets AES-256-GCM ciphertext, zero key material |
| Institute + admin collude | вң… Undetectable | вқҢ Recorded in immutable audit log with IPs |
| Insider tampers with DB directly | вң… May go undetected | вқҢ SHA-256 mismatch detected within 60 seconds |

### TriShield LiveWatch вҖ” Continuous Presence Verification

For every critical paper action (locking or editing), all three parties вҖ” institute, admin, and superadmin вҖ” must be simultaneously present with **active webcam feeds**:

- Webcam frames captured every 10 seconds per party вҶ’ stored in private Supabase Storage
- All three feeds visible simultaneously on all three screens
- **Lock Paper button disabled** until all_parties_present = true
- Superadmin has real-time view of every keystroke during edit window
- **Emergency Halt button** for superadmin to veto any edit in progress
- Session report generated after every action with all snapshot paths, IPs, device fingerprints, and timestamps
- Snapshots auto-purged after 90 days (DPDP Act 2023 compliant), metadata kept permanently
